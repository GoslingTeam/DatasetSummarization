Данные датасета имеют следующую структуру:

**Header**: Первые 16 байт - заголовок, содержащий информацию о формате и размере записи.

**Record**: Затем идут записи данных, каждая из которых состоит из нескольких полей (field). Каждое поле имеет фиксированную длину в битах. 

- **Field 1: SAI (Signature of Automated Identification) - 4 байта** - уникальный идентификатор для автоматического распознавания, используемый в системе.

- ** Field 2: Data Type - 2 байта** - тип данных, который хранится в этом поле (например, дата, текст, число и т.д.).

- ** Field 3: Length of Value - 4 байта** - размер значения для каждого поля в битах. 

- ** Field 4: Value - переменная длина в битах** - само значение для каждого поля.

Обратите внимание, что поле "Value" может иметь разную длину и тип данных, так как это зависит от типа данных и конкретного значения для каждого поля.

Использование фиксированной структуры данных позволяет более эффективно хранить данные на диске или в оперативной памяти и обеспечивает быструю обработку и выборку данных при запросах.
